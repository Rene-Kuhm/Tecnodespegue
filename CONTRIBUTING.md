# Contributing

Thanks for taking the time to improve this project.

## Quick Path

1. Open an issue or describe the change before starting larger work.
2. Create a branch with a clear name.
3. Keep the change focused and easy to review.
4. Update documentation when behavior or setup changes.
5. Open a pull request with a short summary and verification notes.

## Commit Style

Use concise Conventional Commit style when possible:

```text
feat: add new capability
fix: correct broken behavior
docs: improve project documentation
chore: update maintenance files
```

## Review Checklist

- [ ] The change has one clear purpose.
- [ ] Documentation was updated if needed.
- [ ] Setup or verification steps are included in the PR.
